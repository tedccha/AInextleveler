# Deployment to Railway

## Prerequisites
- GitHub account with this repo pushed
- Railway account (free tier available)
- Claude API key

## Deployment Steps

### 1. Create Railway Project
1. Go to [railway.app](https://railway.app)
2. Click "New Project"
3. Select "Deploy from GitHub"
4. Connect your GitHub account and select this repo
5. Railway auto-detects Next.js - continue

### 2. Add PostgreSQL Database
1. In Railway dashboard, click "+ New"
2. Select "Database"
3. Choose "PostgreSQL"
4. Railway automatically creates and connects it

### 3. Configure Environment Variables
In the Railway dashboard, add these variables to your Next.js service:

```
ANTHROPIC_API_KEY=your_claude_api_key_here
DATABASE_URL=<auto-generated by Railway>
NODE_ENV=production
```

The `DATABASE_URL` is automatically set when you add PostgreSQL. You just need to add your Claude API key.

### 4. Run Database Migrations
Railway gives you terminal access. After deployment:
1. Click into your Next.js service
2. Go to "Deployments"
3. Click the active deployment
4. Open "Shell" terminal
5. Run:
```bash
npm run db:push
```

This runs `drizzle-kit push` to create tables and set up pgvector extension.

### 5. Verify Deployment
- Railway shows your app URL (e.g., `your-app.railway.app`)
- Visit it and sign in
- Test by adding a resource

## Environment Variables Reference

| Variable | Value | Notes |
|----------|-------|-------|
| `ANTHROPIC_API_KEY` | Your Claude API key | Get from [console.anthropic.com](https://console.anthropic.com) |
| `DATABASE_URL` | Auto-set by Railway | PostgreSQL connection string with pgvector |
| `NODE_ENV` | `production` | Auto-set to production on Railway |
| `AUTH_PASSWORD` | Set a password | For session auth (set in Railway dashboard) |

## Troubleshooting

### Deployment fails
- Check logs in Railway dashboard (Deployment → Logs)
- Ensure all environment variables are set
- Check that PostgreSQL addon is attached

### Database connection error
- Verify `DATABASE_URL` is set
- Wait 30 seconds after adding PostgreSQL (connection string takes time)
- Check pgvector extension is installed (should be automatic on Railway)

### Authentication issues
- Set `AUTH_PASSWORD` environment variable
- Clear browser cookies and try again
- Check session is working on `/login`

## Costs
- **First $5/month free** on Railway
- PostgreSQL: ~$1-2/month for small databases
- Next.js app: ~$1-5/month depending on compute
- Total: Usually free or very cheap

## Useful Commands

Via Railway terminal:
```bash
# Check database connection
psql "$DATABASE_URL" -c "SELECT version();"

# Run migrations
npm run db:push

# View logs
npm run db:studio  # Prisma Studio (if using Prisma)
```

## Custom Domain
1. In Railway, go to Settings
2. Add custom domain
3. Update DNS records (Railway provides instructions)
4. Takes ~10 minutes to propagate
