# AInextleveler

Personal AI upleveling tool. Local-first Next.js 15 app. Paste X posts / GitHub URLs / text → AI evaluates against your capability profile → routes verdicts and ranks an improvement queue organized by Agentic Lifecycle themes.

## Locked architecture (do not relitigate)

- **Auth:** layout-based gate, Node runtime only. `lib/auth.ts:requireSession()` from `app/(app)/layout.tsx`. iron-session v8, 24h cookie. NO middleware. NO Edge runtime anywhere.
- **DB:** local pgvector via `docker compose up -d` (port 5433). Drizzle with `drizzle-kit push` for v1.
- **Taxonomy:** Agentic Lifecycle — 6 themes × ~4 sub-capabilities = 26 total. `lib/taxonomy.ts` is the single source of truth. Both LLM prompts pin to it.
- **Embedding format:** `${theme} > ${name}` via `capabilityEmbedString()`. Voyage-3, 1024 dims.
- **HNSW indexes** on both vector columns (`lib/db/setup.sql`).
- **GitHub scan:** filter `repo.fork === true` out (simple version v1; full tier system in TODOS).
- **Lesson plans:** `queue_items.lesson_plan: jsonb` with ordered `LessonStep[]`. Generated by Haiku for `not_yet`, by Sonnet for `keep` on Add-to-queue.

## Design system

- **Framework:** Fluent UI (Microsoft design system) via `@fluentui/react-components`
- **Provider:** `FluentProvider` wraps app layout with `teamsLightTheme` for consistent styling
- **Typography:** Fluent default sans-serif (Segoe UI on Windows, -apple-system on macOS)
- **Color palette:** Fluent `teamsLightTheme` (primary: #0078d4 blue)
- **Components:** Use Fluent Button, Card, Input, Textarea, Text, Badge components
- **Inline styles:** Prefer inline styles with Fluent tokens for one-off customizations
